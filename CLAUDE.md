# Trend Watcher Backend

## Project Overview

Trend Watcher backend — a Node.js server running on a Mac Mini at Epilog Creative's office in Jakarta, Indonesia. It scrapes TikTok Explore, scores trends using a 3D scoring engine (Replication Signal, Velocity, Brand Entry Score), and pushes data to Supabase. The Lovable-built React frontend reads from the same Supabase project. This backend never serves the frontend — it only writes data.

Clients: Godrej Indonesia brands — Stella (air freshener), HIT Kecoa (insecticide), NYU (personal care). All trend scoring targets the Indonesian market and Indonesian cultural context.

## Architecture

- **Runtime:** Node.js (CommonJS modules, not ES modules)
- **Scraper:** Playwright with headless Chromium
- **Database:** Supabase via @supabase/supabase-js
- **Scheduler:** node-cron aligned to WIB (UTC+7) peak hours
- **Server:** Express (health check + webhook trigger endpoint only)
- **Notifications:** Slack webhooks via axios
- **AI (future):** OpenRouter for trend analysis calls

## Folder Structure

```
src/
  scrapers/
    tiktok.js           — Playwright scraper for TikTok Explore page. Launches headless
                          Chromium, navigates TikTok, extracts video metadata (title, author,
                          views, likes, comments, shares, hashtags, audio). Exports scrapeOnce().

  scoring/
    engagement.js       — Pure functions to compute engagement_rate (likes+comments+shares)/views
                          and velocity_score from time-series engagement_snapshots.
    replication.js      — Pure functions to estimate replication_count (how many creators copied
                          a format) and compute the Replication Signal score.
    classifier.js       — Pure functions to classify each trend into lifecycle_stage
                          (emerging/growing/peaking/declining/dead) and compute momentum.

  patterns/
    formats.js          — Detects replicable content formats (duets, stitches, transitions,
                          POV skits) from video metadata and hashtag clusters.
    cultural.js         — Identifies Indonesian cultural signals (Ramadan, lebaran, local slang,
                          regional references) for brand relevance scoring.

  database/
    supabase.js         — Initializes Supabase client. Exports upsertTrend() and
                          insertEngagementSnapshot(). Only writes to 'trends' and
                          'engagement_snapshots' tables.

  notifications/
    slack.js            — Sends Slack webhook messages for act-now trends. Formats trend
                          summary, urgency level, and deep-dive link into Slack blocks.

  server.js             — Express entrypoint. Loads dotenv. Mounts GET /health and
                          POST /scrape (behind AUTH_SECRET bearer token). Starts scheduler.

  scheduler.js          — node-cron jobs that trigger scraping runs at Indonesian peak hours
                          (07:00, 12:00, 18:00, 21:00 WIB = UTC+7).

tests/
  scoring.test.js       — Jest unit tests for engagement, replication, and classifier scoring
                          functions with known input/output fixtures.
  patterns.test.js      — Jest unit tests for format detection and cultural signal identification
                          with Indonesian-market test cases.
```

## Environment Variables

| Variable | Purpose |
|----------|---------|
| `SUPABASE_URL` | Supabase project URL (e.g. `https://xyz.supabase.co`) |
| `SUPABASE_ANON_KEY` | Supabase anonymous/public API key for client auth |
| `PORT` | Express server port (default: `3001`) |
| `AUTH_SECRET` | Bearer token required for POST /scrape endpoint |
| `SLACK_WEBHOOK_URL` | Slack incoming webhook URL for trend notifications |
| `NODE_ENV` | `development` or `production` — controls logging verbosity |

Dotenv is loaded once at the top of `src/server.js`. No other file should call `require('dotenv').config()`.

## Coding Conventions

- **CommonJS only** — use `require()` and `module.exports`. Never use `import`/`export`.
- **All async functions must use try/catch** with explicit error logging. Never let a promise reject silently.
- **Never use `console.log`** — use the logger wrapper from `src/logger.js` (create it if it does not exist).
- **All Supabase writes must check for `error`** in the response and log it. Never silently fail on a database write.
- **No hardcoded credentials** — always read from `process.env`.
- **All exported functions must have a JSDoc comment** explaining inputs, outputs, and side effects.
- **Scoring functions must be pure functions** — no side effects, no database calls, no I/O. They take data in and return scores.
- **Never block the event loop** — all I/O must be async. No sync file operations.
- **UUIDs are generated by Supabase** — never generate `id` values in backend code. Let the database default handle it.
- **ISO 8601 timestamps** — use `new Date().toISOString()` for `scraped_at` and `captured_at`.

## Database Rules

### Tables this backend WRITES to:
- **`trends`** — upsert raw scraped trend data. Always upsert using the `url` column (or a hash of it) as the conflict key.
- **`engagement_snapshots`** — insert a new row every time a trend is scraped, capturing views/likes/comments/shares at that point in time.

### Tables this backend NEVER writes to:
- `trend_analysis` — owned by Supabase Edge Functions
- `client_brand_fit` — owned by Supabase Edge Functions
- `team_feedback` — owned by the frontend
- `taste_weights` — owned by Supabase Edge Functions / feedback loop

### Column reference:
All column names must exactly match the TypeScript interfaces defined in:
`../trend-watcher-frontend/src/types/index.ts`

If a column does not exist in those TypeScript types, do not write to it.

## Banned Patterns

- **No sync file operations** — no `fs.readFileSync`, `fs.writeFileSync`, etc.
- **No `process.exit()`** — let errors propagate. The process manager handles restarts.
- **No hardcoded TikTok selectors** outside of the `SELECTORS` constant in `src/scrapers/tiktok.js`.
- **No direct SQL** — always use the Supabase JS client (`supabase.from(...)`).
- **No try/catch that silently swallows errors** — every catch block must log the error.
- **No `console.log` / `console.error`** — use the project logger.
- **No ES module syntax** — no `import`, no `export default`, no `export const`.

## Testing

- **Run tests:** `npm test`
- Scoring functions must have unit tests in `tests/scoring.test.js`.
- Pattern detection must have unit tests in `tests/patterns.test.js`.
- **Do not mock Supabase in tests** — test pure functions only. Scoring and pattern functions should have zero dependencies on external services.
- Test fixtures should use realistic Indonesian TikTok data (hashtags in Bahasa Indonesia, realistic engagement numbers).

## When Adding New Features

1. Read this CLAUDE.md first.
2. Check `../trend-watcher-frontend/src/types/index.ts` to verify column names before writing to Supabase.
3. If the column does not exist in the TypeScript types, do not write to it.
4. If adding a new scraper or data source, follow the same pattern as `src/scrapers/tiktok.js`.
5. If adding a new scoring dimension, add it as a pure function in `src/scoring/` with corresponding tests.
6. Never add middleware or routes that serve HTML — this backend is API-only.
